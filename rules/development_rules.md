# Правила разработки проекта TestDbApp (ZoneHydrantEditor)

1. **Язык:** Весь код, комментарии и документация — на русском (для пользовательского UI) или английском (для технических идентификаторов). Файлы планов и правил — на русском.

2. **Структура проекта:**
   - `Models/` — POCO-модели, мапящиеся на таблицы БД
   - `Helpers/` — сервисы и утилиты (`EwsMapDataService`, `BackupService`, `DatabaseService`)
   - `UserInput/` — диалоговые окна (`AddMarkerWindow`, `BindingEditDialog`)
   - `GraphicElements/` — кастомные GMap-маркеры
   - `SyncPage/` — окно синхронизации/истории
   - `ImageExportWindows/` — экспорт в GridPage
   - `plans/` — файлы планов (.md)
   - `rules/` — файлы правил (.md)

3. **Работа с БД:** Все EWS-таблицы pre-exist в `ews_map.db`. Схема читается через `PRAGMA table_info`. НОВЫЕ колонки добавляются через `ALTER TABLE` в коде (не вручную).

4. **Generic Query:** Используется `Query<T>(tableName)` для SELECT. Для INSERT/UPDATE/DELETE — явный SQL с параметрами.

5. **Принцип единственной ответственности:** Один класс — одна задача. Сервисы не должны знать о UI.

6. **Самопроверка:** Перед завершением задачи проверить:
   - Все пути навигации (куда ведут кнопки, что происходит при нажатии)
   - Нет ли "тупиков" в UI
   - Изменения не нарушают архитектуру
