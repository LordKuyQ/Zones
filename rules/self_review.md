# Правило: Саморецензирование (Self-Review)

Перед завершением любой задачи провести саморецензирование изменений:

1. **Синтаксис:** Нет синтаксических ошибок, опечаток, незакрытых скобок/кавычек.
2. **Логика:** Проверить все пути выполнения — условные операторы, циклы, обработку null.
3. **SQL:** Убедиться, что имена таблиц и колонок соответствуют реальной схеме БД (проверять через PRAGMA).
4. **XAML:** Проверить имена элементов (x:Name совпадает с кодом), Binding-пути.
5. **Нейминг:** Соответствие правилам из `rules/naming_conventions.md`.
6. **Чистота:** Удалить лишние комментарии, `Debug.WriteLine`, неиспользуемые using/imports.
7. **Сборка:** Проект должен компилироваться без ошибок.

**ВАЖНО:** Всегда проверять `PRAGMA table_info` для таблиц, к которым пишется SQL — колонки могут отличаться от ожидаемых.
